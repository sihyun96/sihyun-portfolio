03 publising

GSAP ScrollTrigger를 사용해 메인 화면 스크롤 시 애니메이션이 동작하는 스크롤 효과를 구현.

GSAP ScrollTrigger를 사용해 이미지가 뷰포트에 진입하면 순차적으로 위로 올라오고, 상단에 닿으면 오른쪽으로 이동하는 애니메이션 구현.

GSAP를 사용해 스크롤이 페이지 끝에 닿으면 애니메이션이 실행되도록 구현.

버튼 클릭 시 offsetTop으로 특정 구간으로 부드럽게 스크롤 내리기 구현

scroll 속성으로 특정 구간이 지나면 메뉴 fixed 구현

gsap를 사용해 arrow와 bar에 hover시 움직이는 애니메이션 이벤트 구현

04 problems & solutions

footer 애니메이션이 뷰포트를 벗어나면 처음부터 재시작 하고 싶은데..

사용자가 페이지를 스크롤하여 특정 요소(.bottom-ani)가 뷰포트에 진입하거나 벗어날 때 애니메이션을 실행
뷰포트에서 벗어났을 때와 다시 들어왔을 때 모두 적절한 처리가 필요

-> if문을 이용하여 뷰포트 들어왔을때 애니메이션 실행, 뷰포트 벗어나면 resetAnimation()을 호출하여 애니메이션이 재설정되고 다시 시작할 수 있도록 로직을 수정
-> IntersectionObserver를 설정해 애니메이션의 움직임을 세밀하게 구현함

이미지가 오른쪽으로 이동하면 이미지 뒤에있는 텍스트가 보여야하는데..?

아래에서 위로(y 축) 이동시키는 애니메이션 실행 후 왼쪽에서 오른쪽으로(x 축) 이동시키는 애니메이션 실행
x축 이동시 이미지 section에 backgroud-color 때문에 이미지 뒤에 있는 section 텍스트가 보이지 않음

-> y축 이동하는 애니메이션 다음 backgroundColor: 'transparent' 값을 적용
-> y축 이동시 텍스트가 자연스럽게 나타나도록 딜레이 추가
-> 스크롤이 다음 섹션으로 바로 넘어가지 않도록 빈 애니메이션을 추가하여 스크롤 길이를 연장

main-ani 스크롤 이동시 다른요소에 가려지네?

main-ani 스크롤 이동시 이미지 위로 지나가는 요소들이 있음

-> 이미지를 가리는 구간에서는 z-index: 999를 적용해 이미지가 가장 위에 표시되도록 설정.
-> 이미지가 텍스트를 가리는 구간에서는 opacity 값을 조절하여 자연스럽게 사라지거나 나타나도록 처리.
